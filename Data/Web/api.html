Общая информация

Kot Defense полностью перепрограммируемая игра, в которой каждый игрок может создать своего уникального кота или собаку. В процессе игры вы легко можете добавить специальные баффы или дебаффы, что значительно разнообразит игровой процесс. Кроме того, Kot Defense предлагает возможность менять текстуры и звуки, что позволяет игрокам создавать своих котов или собак по своему желанию. Это даст возможность игрокам создавать совершенно уникальные и запоминающиеся визуальные образы. Также важно отметить, что в игре предусмотрены опции для изменения баланса. Игроки могут корректировать характеристики своих котов и собак, что позволяет адаптировать игру под свои предпочтения и стиль. Это может включать в себя изменение силы атак, количество собак, множители усложнения и другие параметры, что позволяет создавать более сбалансированный и интересный игровой процесс. Особое внимание уделяется финальной волне, которую также можно переделывать, добавляя свои идеи и стратегии для успешного завершения игры. Каждое изменение открывает новые горизонты и возможности, делая каждый игровой опыт по-настоящему увлекательным и непредсказуемым.

Структура дата-пака

Чтобы правильно заменить картинки или звуки в игре, важно следовать правильной структуре папки Data для ваших Data Packs. Вот как должна выглядеть ваша папка Data: 1. Data\DataPacks: - Это основная папка, в которую вы будете добавлять свои Data Packs. 2. \<название дата пака>: - Внутри папки DataPacks создайте папку с названием вашего Data Pack. - Пример: "MyAwesomePack" 3. Картинки и звуки: - Все картинки или звуки, которые вы хотите заменить, помещайте в эту же папку с точно таким же именем, как и у файлов, которые вы хотите заменить. Важно: - Не используйте пробелы в именах файлов! Используйте "_" (нижнее подчеркивание) для разделения слов. - Сохраняйте исходное расширение файла! (например, .png, .mp3). Пример: - Если вы хотите заменить картинку "play.png" в игре, то создайте папку "MyPackName" в "Data\DataPacks" и поместите туда свою новую картинку.

Как работает дата-пак

Он проходит по папкам с Data Pack, ищет там нужные файлы, например, картинки `back.png` и `btn.png`. Потом он их загружает в игру и заменяет новый файл на старый. Также есть файл `theme.py`, который содержит специальные инструкции для игры. Скрипт читает этот файл, и выполняет все эти инструкции для изменений цвета и тд.

Своя тема

Тема - это набор цветов, которые определяют внешний вид игры. Она хранится в файле `theme.py` в Data Pack. В этом файле ты можешь менять цвета текста, кнопок, фона и других элементов игры, чтобы создать свой уникальный стиль. Как создавать тему: 1. Создай файл `theme.py` в папке твоего Data Pack. 2. В файле `theme.py` задай значения для цветов: - Используй имена переменных, которые уже есть в игре, например: 


text_color = (255, 255, 255)
text_outlining_color = (0, 0, 0)
# ... и так далее

- Замени значения цветов на свои, используя цветовой код RGB (три числа от 0 до 255, которые представляют красную, зеленую и синюю составляющие цвета). Где использовать тему: - В папке Data Pack: Файл `theme.py` должен находиться в папке с Data Pack. - В игре: Скрипт, который ты мне прислал, будет автоматически загружать и использовать цвета из файла `theme.py` при запуске игры.

Пример темы


text_color = (100, 100, 100)  # Серый текст
text_outlining_color = (50, 50, 50) # Более темный серый контур текста
text2_color = (150, 150, 150) # Более светлый серый текст
heart_color = (255, 0, 0)   # Красный цвет сердечек
money_color = (255, 255, 0) # Желтый цвет монет
background_color = (0, 0, 0)   # Черный фон
event1_background_color = (50, 50, 50) # Серый фон для события 1
red_btn_color = (255, 0, 0)  # Красный цвет кнопки
green_btn_color = (0, 255, 0) # Зеленый цвет кнопки
blue_btn_color = (0, 0, 255) # Синий цвет кнопки
down_menu_cats = (100, 100, 100) # Серый цвет меню
event1_down_menu_cats = (50, 50, 50) # Более темный серый цвет меню для события 1
info_text_outlining_color = (50, 50, 50) # Серый контур для информационного текста
info_text2_color = (200, 200, 200) # Светло-серый цвет информационного текста
skip_button_color = (255, 255, 0) # Желтый цвет кнопки пропуска
skip_button_text_color = (0, 0, 0) # Черный цвет текста на кнопке пропуска
boss_background_color = (150, 0, 0) # Темно-красный фон для босса

Структура мода

Data\Mods\<Название мода>\ В этой папке хранятся все файлы мода. Список необходимых папок: - Data: Эта папка содержит все основные файлы мода. - Data/Img: Здесь хранятся изображения для мода, например, текстуры для новых персонажей, предметов или спрайты. - Data/Songs: В этой папке хранятся музыкальные файлы для мода, такие как фоновая музыка, звуковые эффекты. - Data/Saves: Папка для хранения сохранений мода, если они используются. - Data/Fonts: Здесь хранятся шрифты, которые используются в моде. - Data/Scripts: В этой папке хранятся все скрипты мода, написанные на Python. Файлы: - main.py: Этот файл будет запускаться каждый раз при запуске игры. В нем следует писать код, который создает новые объекты, добавляет новые элементы в игру и т.д. - load.py: Этот файл запускается только один раз при запуске игры. В нем можно писать код, который инициализирует мод, загружает ресурсы, настраивает игровые параметры и т.д.

Важно

Глобальные переменные: Чтобы получить доступ к переменной из игры или применять её в другой части кода, необходимо использовать ключевое слово `global <название переменной>` в начале строки кода. Доступ к ресурсам мода: Для доступа к картинкам, звукам и другим файлам мода необходимо использовать абсолютный путь от корневой папки игры. К примеру: 


Data\Mods\<Название мода>\Data\Songs\<Название звука и формат>
Data\Mods\<Название мода>\Data\Img\<Название картинки и формат>
Важно что "\<Название мода>\" – это имя вашей папки мода. А так же важно в любом пути перед " НЕОБХОДИМО писать r то есть r"Data\Mods\<Название мода>\Data\Songs\<Название звука и формат>", это так важно потому-что python может просто не понять что это путь и не найдёт файл

Подключение скриптов

Подключение скриптов в играх и приложениях - это процесс загрузки и выполнения внешних файлов кода, которые могут изменять игровой процесс, добавлять новые функции, персонажей, предметы и т.д. Основные этапы подключения скриптов: 1. Загрузка скриптов: * Чтение файла: С помощью функций `open()` и `read()` прочитайте содержимое файла скрипта в строку. * Обработка кода: * Проверка безопасности: Проверьте код скрипта на наличие потенциально опасных элементов (например, встроенные функции, которые могут выполнять вредные действия). * Синтаксический анализ: Проверьте код на синтаксические ошибки, чтобы убедиться в его корректности. * Преобразование в исполняемый код: Преобразуйте прочитанный текст в исполняемый код. 2. Выполнение скриптов: * Использование `exec`: Функция `exec(код)` выполняет указанный код в текущей области видимости. * Использование `eval`: Функция `eval(выражение)` выполняет указанное выражение и возвращает результат. * Использование модуля `import`: Если скрипт содержит определения функций и классов, можно импортировать его с помощью `import имя_модуля`. Пример: Представьте, что нужно подключить скрипт `character_script.py`, который добавляет новое поведение котв на получение урона. Мы пишем в load.py вот такие строчки: 


Script(r"Data\Mods\<Название мода>\Data\Scipts\character_script.py", "bump_cat") # Script(<Путь к скрипту>, < ID в виде названия в какой части игры надо его запустить>)
Все нужные "ID в виде названия в какой части игры надо его запустить" можно найти в исходном коде на гит хабе.

Они все выглядят примерно так: 


for script in scripts:
	exec(str(start_script(script, ID который отсылает на эту строку, то есть ваш скрипт запуститься на месте этой строки)))

Kot Defense Maker

Kot Defense Maker — это простой и удобный инструмент для создания модификаций, таких как новых котов и собак. Этот интерфейс позволяет пользователям легко вводить необходимые параметры без необходимости написания кода. Преимущества: - Удобный интерфейс: Пользователи могут легко навигировать по программному обеспечению и не нужно быть программистом, чтобы создать своего персонажа. - Графическое представление: Визуальное представление всех характеристик персонажа, что упрощает восприятие и корректировку параметров. - Интуитивно понятное управление: Пользователи могут выделять и изменять параметры с помощью простых элементов управления, таких как ползунки и выпадающие списки. - Предпросмотр в реальном времени: В Kot Defense Maker предусмотрена функция предварительного просмотра, которая позволяет пользователю увидеть, как кот или собака будет выглядеть в игре. Это помогает быстро оценить внешний вид персонажа, его анимацию и общий стиль. Недостатки: - Ограниченный функционал: Несмотря на простоту использования, Kot Defense Maker может иметь менее сложные возможности по сравнению с написанием кода вручную, что может ограничить костамизацию для продвинутых пользователей. - Нельзя сделать все: Некоторые специфические функции и параметры могут быть недоступны в интерфейсе, что затрудняет создание очень уникальных или сложных персонажей. В заключение, Kot Defense Maker идеально подходит для людей, которые хотят быстро и без затруднений создавать котов и собак для своей игры, но у него есть свои ограничения в плане глубины функционала.

Создание новых котов

Создание кота в игре не требует обширных знаний или навыков. Вам необходимо создать класс с описанием его характеристик. Вот пример класса, который отвечает за создание кота: 


class BuyCat:
    def __init__(self, moneys, info_self, aura_radius, cat_health, dogs_count, reload, damage_cat, mega, wawe, wawedel, path_my_image, size_x, size_y, ico_image_path):
        self.moneys = moneys  # Цена кота
        self.info_self = info_self  # Информация о коте
        self.aura_radius = aura_radius  # Радиус воздействия
        self.cat_health = cat_health  # Здоровье кота
        self.dogs_count = dogs_count  # Максимальное количество атакуемых собак
        self.reload = reload  # Время перезарядки
        self.damage_cat = damage_cat  # Урон, наносимый котом
        self.mega = mega  # Уникальность кота
        self.wawe = wawe  # Появление на волне
        self.wawedel = wawedel  # Пропадание на волне (-1 означает бесконечное присутствие)
        self.size_x = size_x  # Ширина изображения кота
        self.size_y = size_y  # Высота изображения кота
        self.ico_image_path = ico_image_path  # Путь к изображению иконки кота, но можно установить None — это значит иконка кота будет как и на поле боя

Описание элементов класса: - Цена (Price): moneys — количество монет, необходимых для покупки кота. - Информация (Information): info_self — текстовое описание кота. - Радиус воздействия (Radius of damage): aura_radius — радиус, в котором кот может наносить урон. - Здоровье (Health): cat_health — общее количество здоровья кота. - Максимальные атаки собак (Maximum dog attacks): dogs_count — максимальное число собак, которые были атакуемы этим котом. - Перезарядка (Reload): reload — время ожидания между атаками. - Урон (Damage): damage_cat — количество урона, который наносит кот врагам (Собакам). - Уникальность (Uniqueness): mega — параметр, определяющий уникальные качества кота (На данный момент существует от 1 до 8 Id уникальностей). - Появление на волне (Will appear on wave): wawe — номер волны, на которой кот появится. - Пропадание на волне (Disappears on wave): wawedel — номер волны, на которой кот исчезнет (-1 означает, что кот будет у вас навсегда). - Размер по оси X (Size x): size_x — ширина изображения кота в пикселях. - Размер по оси Y (Size y): size_y — высота изображения кота в пикселях. - Путь к изображению иконки (Icon image path): ico_image_path — путь к изображению иконки кота, но можно установить None — это значит иконка кота будет как и на поле боя Таким образом, создание класса для кота в игре — это простой и удобный способ задать все необходимые характеристики и свои уникальные свойства.

Информация о уникальностях котов

Уникальности позволяют разнообразить игровой процесс и учитывать различные стратегии в зависимости от выбранных котов. Каждый кот приносит свои уникальные способности, которые могут сильно повлиять на исход боя. id 1: Этот тип кота не обладает особыми способностями, но его присутствие дает возможность команде действовать, как обычно. Он служит базовым элементом в игре, сосредотачиваясь на стандартных атаках. id 2: Этот кот приносит {self.damage + cat_works_boost}$ за волну в место удара. Он помогает игроку накапливать ресурсы, что позволяет улучшать других котов и постепенно укреплять оборону. id 3: Кот этой категории усиливает удар курсором на величину self.damage. Это позволяет игроку наносить значительно больший урон врагам, улучшая эффективность боя в критические моменты. id 4: Этот кот восстанавливает всем котам в его радиусе действия health на величину self.damage. Это делает его незаменимым во время длительных боев, где необходимо поддерживать здоровье команды на высоком уровне. id 5: Кот, который при получении урона наносит self.damage% урона собаке, которая его ударила. Эта уникальность позволяет ему не только защититься, но и отомстить своим обидчикам, внося дополнительный урон. id 6: Каждую волну этот кот спавнит жабу, которая не очень сильная, но может добавлять разнообразие в бой. Это помогает в защите или атаке. id 7: Каждую волну спавнит добрую собаку, что добавляет новой динамики в игру. Эта собака будет помогать команде, отвлекая противников и создавая дополнительное давление на врага. id 8: Атакует всех собак в радиусе действия, нанося им урон, и одновременно наносит self.damage/500 всем котам в радиусе действия. Это позволяет одновременно вести атаку на врагов и ослаблять своих котов.

blacklist и damagecats

кот может ударить собаку только в том случае, если список `blacklist` кота совпадает со списком `damagecats` собаки. Вот как это должно работать: 1. Проверка на атаку: - Игра проверяет список `damagecats` собаки. - Игра проверяет список `blacklist` кота. 2. Условия атаки: - Если списки совпадают (содержат одинаковые типы собак), то кот может атаковать собаку. - Если списки не совпадают, то кот не может атаковать собаку. Пример: - Собака: Тип 0 (Обычная собака), `damagecats` = [0, 1]. - Кот: `blacklist` = [0, 1]. В этом случае кот может атаковать собаку, потому что их списки совпадают. - Собака: Тип 1 (Собака взрывающаяся), `damagecats` = [0]. - Кот: `blacklist` = [1, 2]. В этом случае кот не может атаковать собаку, потому что их списки не совпадают. ps простите я кажись в хлам был когда писал этот код...

Как добавить blacklist

Представим что у нас есть кот. 


new_cat = BuyCat(
    moneys=200,                 # Цена: 100 монет
    info_self="Кот сапёр",      # Информация о коте
    aura_radius=500,            # Радиус воздействия: 300
    cat_health=10,              # Здоровье: 50
    dogs_count=1,               # Максимальные атаки собак: 2
    reload=5,                   # Время перезарядки: 2 секунды
    damage_cat=500,             # Урон кота: 15
    mega=1,                     # Уникальность: 1, обычный атакующий кот 
    wawe=0,                     # Появление на волне: 0, значит сразу в начале игры
    wawedel=-1,                 # Пропадание на волне: бесконечно
    path_my_image=r"Data/Img/Cat5.png", # Путь к оригинальному изображению кота
    size_x=80,                  # Ширина: 50 пикселей
    size_y=80,                  # Высота: 70 пикселей
    ico_image_path=None         # None - значит иконка кота будет как и на поле боя
)

Чтобы кот бил только взрывающихся собак, необходимо добавить тип взрывающейся собаки (id 1) в список `blacklist` кота. Это значит, что кот не сможет атаковать все типы собак, кроме взрывающихся. Проще говоря, нужно добавить тип взрывающейся собаки в список запрещенных для атаки кота. Пример: 


new_cat = BuyCat(
	#параметры
)
new_cat.blacklist.append(1) # добавляем в список id 1

Пример создания своего кота

Для создания экземпляра класса BuyCat мы можем использовать следующий код: 


new_cat = BuyCat(
    moneys=100,                 # Цена: 100 монет
    info_self="Сильный кот",    # Информация о коте
    aura_radius=300,              # Радиус воздействия: 300
    cat_health=50,              # Здоровье: 50
    dogs_count=2,               # Максимальные атаки собак: 2
    reload=2,                   # Время перезарядки: 2 секунды
    damage_cat=15,              # Урон кота: 15
    mega=1,                  # Уникальность: 1, обычный атакующий кот 
    wawe=0,                     # Появление на волне: 0, значит сразу в начале игры
    wawedel=-1,                 # Пропадание на волне: бесконечно
    path_my_image=r"Data/Img/Cat2.png",  # Путь к оригинальному изображению кота
    size_x=50,                  # Ширина: 50 пикселей
    size_y=70,                  # Высота: 70 пикселей
    ico_image_path=None   # None - значит иконка кота будет как и на поле боя

Создание новых собак

Создание собаки в игре не требует обширных знаний или навыков. Вам необходимо создать класс с описанием его характеристик. Вот пример класса, который отвечает за создание собаки: 


class Dog:
    def __init__(self, money_dog, info_self, aura_radius, health_dog, cats_count, reload, damage_dog, mega, wave, wawedel, size_x, size_y, ico_image_path):
        self.money_dog = money_dog
        self.info_self = info_self
        self.aura_radius = aura_radius
        self.health = health_dog
        self.cats_count = cats_count
        self.reload = reload
        self.damage = damage_dog
        self.mega = mega
        self.wave = wave
        self.wawedel = wawedel
        self.size_x = size_x
        self.size_y = size_y
        self.ico_image_path = ico_image_path

Описание элементов класса Собака: - Деньги собаки (Money dog): money_dog — представляет количество денег, которые игрок получает за уничтожение этой собаки. - Бюджет (Bugete): bugete — представляет стоимость собаки, для покупки этой собаки у игры (система спавна собак функционирует на основе бюджета, выделяемого игре. Игра использует эти деньги для покупки и размещения собак на поле боя.). - Имя (Name): name — уникальное название собаки. Оно служит идентификатором собаки в игре, позволяя легко ссылаться на неё и отличать её от других собак, если имена собак в некоторых модах будут одинаковы это может превести к сбою. - Здоровье собаки (Health dog): health_dog — это общее количество здоровья собаки. Этот параметр определяет, насколько устойчива собака к урону, получаемому в битвах или других игровых ситуациях. - Скорость (Speed): speed — это скорость, с которой собака может перемещаться по игровому полю. Она влияет на передвижение собаки и может быть важна в зависимости от стратегий игры. - Урон собаки (Damage dog): damage_dog — это количество урона, которое собака наносит противникам. Этот параметр является ключевым для определения боевой эффективности собаки. - Уникальное свойство (Unikal): unikal — это характеристика, которая определяет уровень уникальности собаки. Она может обозначать специальные способности или модификации, отличающие одну собаку от другой. - Время перезарядки (Reload): reload — это временной интервал, который собака должна ждать, прежде чем сможет снова атаковать. Этот параметр влияет на темп боя и комбинацию атак собаки. - Тип собаки (Type dog): type_dog — это категория собаки, указывающая на её роль в игре, например, охотничья, служебная или другая. Тип может влиять на поведение и способности собаки. - Путь к изображению (Path_my_image): pathmyimage — это адрес, где хранится изображение собаки в файловой системе. Этот путь используется для визуального отображения собаки в интерфейсе игры. - Ширина (Size x): size_x — это ширина изображения собаки в пикселях. Она помогает определить размер собаки на экране. - Высота (Size y): size_y — это высота изображения собаки в пикселях. Этот параметр также участвует в определении визуального размера собаки на игровом поле. Таким образом, создание класса для собаки в игре — это простой и удобный способ задать все необходимые характеристики и свои уникальные свойства.

Уникальности собак

Уникальности собакам в игровом контексте необходимы для создания разнообразия и глубины игрового процесса. id 1: Обычная собака. Атакует при столкновении с котом. id 2: Взрывающаяся собака. Взрывается при столкновении с котом в радиусе 200px. id 3: Спавнит слабых собак. При столкновении или по истечении времени спавнит несколько слабых собак по бокам. id 4: Движется вверх. При появлении движется вверх на 60px. id 5: Движется вниз. При появлении движется вниз на 60px. id 6: Спавнит взрывающиеся собаки и обычные. При ударе спавнит одну взрывающуюся собаку по бокам; при смерти — 3 обычные собаки и 2 взрывающиеся. id 7: Взрывающаяся собака, движущаяся вверх. При появлении движется вверх на 60px. id 8: Взрывающаяся собака, движущаяся вниз. При появлении движется вниз на 60px. id 9: Проходит сквозь котов. Может использоваться для обхода защитников. id 10: Спавнит обычные и взрывающиеся собаки. После смерти спавнит 3 обычные и 2 взрывающиеся собаки, а также собаку "мехо" с ID 6. id 11: Собака-волк. После смерти становится волком и восстанавливает всё здоровье. id 12: Лечащая собака. Лечит всех собак в радиусе 200px на self.damage единиц. Если моды используют одинаковые ID (именно использования своего ID для уникального функционал собаки), это может привести к серьезным проблемам и конфликтам. Советуем создавать свои ID только продвинутым пользователям

Типы собак

Типов собак не так и много, и они используются в качестве информации для котов. id 0: обычная собака id 1: собака взрывающаяся id 2: собака летающая

Как работают переводы

В этой документации описывается, как работает система переводов в вашей игре. Переводы хранятся в двух словарях, которые позволяют легко получать текст на нужном языке в зависимости от текущих настроек пользователя. Переводы хранятся в словаре translate_text, который имеет следующую структуру: 


translate_text = {
    "ru": {
        "debaff_hard": "игра станет сложнее",
        "debaff_all_cats_damage": "некоторым котам на поле боя нанесут 10 урона",
        "debaff_maxium_money": "меньше на максимальное число денег которое у вас было"
		# И так далее
    },
    "en": {
        "debaff_hard": "game will become more difficult",
        "debaff_all_cats_damage": "some cats on the battlefield will be dealt 10 damage"
		# И так далее
    }
}

На данный момент поддерживаются следующие языки: - Русский (ru) - Английский (en) Для получения перевода необходимо выполнить следующие шаги: 1. **Получить глобальные переменные**: Убедитесь, что вы имеете доступ к глобальным переменным translate и translate_text. Переменная translate должна содержать текущий язык, выбранный пользователем. 2. **Использование словаря для получения перевода**: Для получения перевода текста используйте следующую конструкцию: 


translated_text = translate_text[translate]["debaff_hard"]

Предположим, что текущий язык установлен на русский (ru). Чтобы получить перевод текста "игра станет сложнее", вы можете использовать следующий код: 


global translate, translate_text
translate = "ru"  # Текущий язык
translated_text = translate_text[translate]["debaff_hard"]
print(translated_text)  # Вывод: игра станет сложнее

Если текущий язык установлен на английский (en), то код будет выглядеть так: 


global translate, translate_text
translate = "en"  # Текущий язык
translated_text = translate_text[translate]["debaff_hard"]
print(translated_text)  # Вывод: game will become more difficult

Система переводов в вашей игре проста и эффективна. Используя предложенные подходы, вы можете легко добавлять новые языки и тексты для перевода, обеспечивая тем самым удобство для игроков из разных стран.

Как создать свой перевод

В этой секции вы узнаете, как добавить новый перевод в вашу игру и создать собственный язык перевода. Это позволит вам расширить аудиторию игроков и сделать игру доступной для большего числа пользователей. Шаги по добавлению нового языка перевода: 1. Определите новый язык Прежде всего, выберите язык, который вы хотите добавить. Для примера, давайте добавим испанский язык (es). 2. Обновите словарь translate_text Добавьте новый ключ для вашего языка в словарь translate_text. Вам нужно будет указать переводы для всех текстов, которые вы хотите локализовать. Например: 


translate_text = {
    "ru": {
        "debaff_hard": "игра станет сложнее",
        "debaff_all_cats_damage": "некоторым котам на поле боя нанесут 10 урона",
        "debaff_maxium_money": "меньше на максимальное число денег которое у вас было"
    },
    "en": {
        "debaff_hard": "game will become more difficult",
        "debaff_all_cats_damage": "some cats on the battlefield will be dealt 10 damage"
    },
    "es": {  # Новый язык
        "debaff_hard": "el juego se volverá más difícil",
        "debaff_all_cats_damage": "algunos gatos en el campo de batalla recibirán 10 de daño",
        "debaff_maxium_money": "menos del número máximo de dinero que tenías"
    }
}

3. Установите текущий язык Теперь, когда вы добавили новый язык в словарь, вы можете установить его как текущий язык. Например: 


global translate
translate = "es"  # Устанавливаем испанский как текущий язык

4. Получение перевода Используйте тот же метод, что и раньше, чтобы получить перевод для нового языка: 


translated_text = translate_text[translate]["debaff_hard"]
print(translated_text)  # Вывод: el juego se volverá más difícil

Как сделать через мод

Чтобы добавить новый язык к существующему словарю или добавить новое слово и его значение в уже существующий язык, вы можете использовать следующие шаги: 1. Добавление нового языка Если у вас есть словарь переводов, вы можете просто добавить новый ключ для языка и его значения. Например, если у вас есть словарь с русским и английским языками, и вы хотите добавить испанский: 


translate_text = {
    "ru": {
        "debaff_hard": "игра станет сложнее",
        "debaff_all_cats_damage": "некоторым котам на поле боя нанесут 10 урона"
    },
    "en": {
        "debaff_hard": "game will become more difficult",
        "debaff_all_cats_damage": "some cats on the battlefield will be dealt 10 damage"
    }
}

# Добавляем испанский язык
translate_text["es"] = {
    "debaff_hard": "el juego se volverá más difícil",
    "debaff_all_cats_damage": "algunos gatos en el campo de batalla recibirán 10 de daño"
}

2. Добавление нового слова в существующий язык Если вы хотите добавить новое слово в уже существующий язык, просто добавьте новую пару ключ-значение в соответствующий словарь. Например, добавим новое слово в испанский и английский языки: 


# Добавляем новое слово в испанский
translate_text["es"]["debaff_maxium_money"] = "menos del número máximo de dinero que tenías"

# Добавляем новое слово в английский
translate_text["en"]["debaff_maxium_money"] = "less than the maximum amount of money you had"

4. Получение перевода Используйте тот же метод, что и раньше, чтобы получить перевод для нового языка: 


translated_text = translate_text[translate]["debaff_hard"]
print(translated_text)  # Вывод: el juego se volverá más difícil
